支払いが、体験の一部になる。

Tempo の MPP 決済とつないだ、超シンプルな課金デモLPです。説明は短く、体験は強く。 トップページでは価値を見せ、実際の受け取りは /api/celebrate 側で実行します。

/api/celebrate へ接続して結果を表示します。

何が起きる？

未決済状態では 402 が返る場合があります。決済後は「お支払いありがとうございます！」文言付き JSON が返却されます。

GET /api/celebrate

なにがいい？

公開ページはシンプルに保ち、決済ロジックはAPI側に閉じるため安全性と保守性を両立できます。

payment middleware + gateway

設定が必要なもの

受け取り先と通貨アドレスを Vercel 環境変数で定義します。

TEMPO_RECIPIENT / TEMPO_CURRENCY

デプロイ

GitHub 連携済みなら、main への push で本番配信されます。ローカルでの再現テストも可能です。

git push origin main
これは課金付き API の導入を可視化するサンプルLPです。トップは見せるだけ、実決済はエンドポイントで。
